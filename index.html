Problem 1: Inverted Number Pyramid

In this first problem, your program will take a single positive number as a command line argument (an integer between 0 and 9 inclusive). Your program should then produce an inverted pyramid-like structure with a number of rows equal to that number. The first row should have as many numbers as that number, with the first row consisting of 1's, the second row consisting of 2's and so on, with the length of each row decreasing by one each time.

Input

The program should take as a command line argument an integer from 0 to 9. For example:

prompt$ prog1 5

Output

Your program should output the inverted pyramid with no extra ending spaces and each line should end with an endline.

Problem 2: Compute Pairwise Distance maps from Points in 3D Space

You are given the x, y, and z coordinates of N number or points in a 3D coordinate system. Your task is to compute Euclidian distance between each point with every other point and print it in the form of a table (usually known as distance map).

Input

The input to your program is a text file with three columns and N rows, space delimited. Each row represents a point in the 3D space. The three columns are the x, y, and z coordinates of the corresponding point. For example, if we wanted to run it using the input file input3d.txt, it should run as follows:

prompt$ prog2 input3d.txt

Output

The output of your program will be a distance map (a 2D matrix) of size N by N written to stdout. The order of the distance matrix should follow the order of the input points. For example, if we consider the first row as point 1, the first cell in the distance matrix should correspond to the distance between point 1 and itself. Similarly, the cell (5, 7) should correspond to the distance between points 5 and 7. This output should be rounded down to a single decimal digit of precision.

Problem 3: Motel Charges

Write a program to properly compute the cheapest rate a person can get a room for. The rules are as follow. Discounts add up unless explicitly prohibited.

No validation needs to be performed.


AAA gets you 10% discount. But if you are under 19 this is only 5%, and over 65 gets you 15%.

If you are 60 or older you get extra 10% discount if you are resident of the state and 5% if you are resident of today's friendly state.

Military veterans get extra 10% but this cannot be combined with any AAA discount.

If you are paying with a credit card, you get a 10% discount but only if you are a resident of today's friendly state.

Input

The input is today's charge (0.01+), local state (US state), today's friendly state (US state), person's age (1+), AAA membership (AAA/No), military veteran status (Vet/No), state residency (US state), paying by credit card (Yes/No). All data supplied through the command line.


For example, suppose that:

Today's charge is 79.99 and it is Missouri. Friendly state is AK. The person is age 55, AAA member but not military vet residing in Missouri and paying with credit.

The input for this scenario would be:

prompt$ prog3 79.99 MO AK 55 YES NO MO YES

Output

The output will be the cheapest rate truncated to the penny (not rounded). For example, 56.78 is a valid rate.
